# 安全事件应急响应

## 适用范围
账号被盗、异常登录、API凭证泄露、大量异常调用等安全事件。

## SOP
1. 查询账号状态 — 确认是否已自动冻结 (LOCKED/SUSPENDED)，风险标记状态
2. 查询最近的登录/操作日志（从工单或系统记录中获取）
3. 确认安全团队的处理进度（关联工单）
4. 输出事件摘要、当前处置状态和客户应采取的后续措施

## 领域规则
- 安全事件优先级 P0，30分钟内必须响应
- 账号自动冻结 → 系统已触发防护，无需手动冻结
- 凭证泄露 → 强制重置所有密钥和密码
- 异常 API 调用 → 记录来源 IP 和调用模式
- 客户需配合提供: 近期是否有可疑操作/邮件

## 提示词模板
账号 {accountId} 疑似发生安全事件。请查询账号当前状态、风险标记、关联工单和安全团队的处置进展。

## 输出规范
- 先告知当前账号状态（是否已安全冻结）
- 简要说明事件经过
- 告知安全团队处理进度
- 给出客户需配合的操作建议
- 语气要安抚但专业

## 常见场景
| 场景 | 触发原因 | 处置 |
|------|----------|------|
| 账号LOCKED | 异常登录检测 | 安全团队分析后解冻 |
| API滥用 | 凭证泄露 | 重置密钥+排查泄露源 |
| 多点异常登录 | 密码泄露 | 强制改密+开启MFA |
