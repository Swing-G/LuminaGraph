# 权限异常排查

## 适用范围
用户反馈无权限访问某个模块/功能，提示 403/权限不足/无权操作等。包括批量权限异常、新员工权限、角色变更后异常。

## SOP
1. 查询账号基本信息 — 确认账号状态是否 ACTIVE（非冻结/过期）
2. 查询订阅状态 — 确认订阅是否过期导致功能受限
3. 查询关联工单 — 确认是否已有权限相关的处理记录
4. 输出根因和解决建议 — 区分订阅过期 vs RBAC配置 vs LDAP同步 vs 角色变更

## 领域规则
- 账号状态 LOCKED → 安全事件，需安全团队介入
- 订阅状态 PAST_OUT/SUSPENDED → 功能自然失效，非权限问题
- 批量权限异常 → 优先排查角色绑定或策略变更
- 新员工权限缺失 → 检查 LDAP 同步或入职流程
- 企业内部系统权限变更需保留审计日志

## 提示词模板
用户 {accountId} 反馈无法访问系统功能。请完整查询账号状态、订阅状态和相关工单，判断根因并给出处理建议。

## 输出规范
- 明确告知根因是订阅过期还是权限配置
- 如有处理中工单告知进度
- 给出具体负责团队和预计解决时间

## 常见场景
| 场景 | 根因 | 处理 |
|------|------|------|
| 全体404/403 | 订阅过期 | 检查订阅状态 |
| 部分人403 | RBAC策略误配 | 检查角色绑定 |
| 新员工403 | LDAP同步延迟 | 手动触发同步 |
| 突然403 | 角色被变更 | 审计角色变更记录 |
