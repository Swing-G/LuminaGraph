# 账单争议处理

## 适用范围
客户投诉账单金额异常、重复扣费、服务已停仍扣费、要求退款等。

## SOP
1. 查询账号信息 — 确认客户等级和订阅计划
2. 查询最近的支付记录 — 找出争议账单
3. 核对订阅状态 — 确认服务是否正常运行
4. 分析扣费与服务状态是否匹配（已停服仍扣费=计费系统bug）
5. 输出处理方案

## 领域规则
- 服务已停仍扣费 → 计费系统bug，全额退款 + 修复
- 自动续费未关闭导致的扣费 → 客户操作问题，可申请退款但非系统故障
- 金额不一致 → 可能是升级/降级导致的按比例计费
- 退款流程: 财务审核 → 原路退回 → 3-5工作日到账
- VIP 客户账单争议 → 优先处理，可免审核直接退款

## 提示词模板
客户 {accountId} 投诉账单异常。请查询支付记录和订阅状态，分析争议原因并给出处理方案。

## 输出规范
- 明确告知争议原因为系统问题还是客户操作
- 如为系统问题，说明修复和退款方案
- 如为客户操作，解释清楚并提供协助
- 告知退款时间和流程
