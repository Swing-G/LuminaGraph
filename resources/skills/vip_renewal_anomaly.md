# VIP客户续费异常处理

## 适用范围
VIP / 企业版客户反馈续费失败、订阅过期、权限受限等问题。

## SOP (标准处理流程)
1. 查询客户账号状态 — 通过 accountId 查询账号基本信息（状态、级别、风险标记）
2. 查询订阅状态 — 确认订阅是否 PAST_OUT / 过期时间 / 自动续费是否开启
3. 查询最近支付记录 — 定位最近一次扣款，确认失败原因（重点关注风控拦截）
4. 查询关联工单 — 确认是否已有处理中的售后工单
5. 综合输出处理建议和客户回复话术

## 领域规则
- **VIP 客户 SLA**：2小时内必须给出处理方案
- **风控拦截**：扣款失败原因为 "RiskEngine: 异常设备多点登录" 时，需联系合规团队加白，不可直接让客户重试
- **自动续费失败**：订阅状态 PAST_OUT 但 auto_renew_enabled=true 时，根因在支付环节而非配置问题
- **工单去重**：如已有 OPEN/PROCESSING 状态工单，告知当前处理进度，不要建议重新创建
- **客户安抚**：VIP 客户需表达重视，明确给出解决时间和责任人

## 提示词模板
```
客户 {accountId} 反馈 {问题描述}。
请按以下步骤完整排查：
1. 查询账号 {accountId} 的状态和风险标记
2. 查询订阅状态和自动续费配置
3. 查询最近的支付记录和失败原因
4. 查询是否已有处理中的工单
5. 基于以上信息，输出：根因分析、当前处理进度、给客户的回复话术、给内部团队的建议
```

## 输出规范
- ✅ 用客服口吻，直接给出答案，不要带分析过程
- ✅ 先结论后细节
- ✅ 如已有工单，告知工单号和当前处理人
- ❌ 不要使用 "根据查询结果" 这类开场白
- ❌ 不要在 VIP 场景下建议客户自行排查

## 常见场景速查
| 场景 | 根因 | 处理 |
|------|------|------|
| 订阅过期+自动续费开启+支付失败 | 风控拦截 | 合规加白→手动重试 |
| 订阅过期+自动续费关闭 | 未开启续费 | 建议开启+手动续费 |
| 订阅正常+权限异常 | 角色配置问题 | 检查RBAC配置 |
| 已有P1工单+客户追问 | 处理中 | 告知进度+加急 |
